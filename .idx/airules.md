# HypGo — Gemini Code Assist Rules

完整規範：`config/AI_CODING_RULES.md`

## 核心規則

1. **所有 func 必須帶 `@ai` 註解區塊**
   - Exported func 需含：`@ai:generated` / `@ai:purpose` / `@ai:input` / `@ai:output` / `@ai:sideeffect`
   - Unexported func 至少需 `@ai:generated`

2. **日期格式**：`YYYY-MM-DD`（不接受其他格式）
   **模型名稱**：`gemini-2.5-pro` 或官方識別符

3. **Schema-first 路由**：
   - REST：`r.Schema(schema.Route{...}).Handle(handler)`
   - gRPC：`schema.RegisterGRPC("ServiceName/MethodName", summary, input, output)`
   - Bot / MCP / WebSocket / CLI 各有對應 helper

4. **錯誤處理**：使用 `pkg/errors` typed catalog，不直接 `c.JSON(500, ...)`

5. **DB 存取**：`hidb.ReadHypDB()` 讀、`hidb.WriteHypDB()` 寫

## 函式註解模板

```go
// FuncName <一句話描述，動詞開頭>
//
// @ai:generated by=gemini-2.5-pro date=YYYY-MM-DD
// @ai:purpose <為什麼需要這個函式>
// @ai:input <參數語意，不是型別>
// @ai:output <回傳語意與狀態碼>
// @ai:sideeffect <副作用 或 none>
func FuncName(...) {}
```

## 禁止事項

- ❌ `panic()` 於 handler
- ❌ `fmt.Println` 做日誌（用 `pkg/logger`）
- ❌ `math/rand` 產生 token（用 `crypto/rand`）
- ❌ 硬編碼 DSN / API key

完整規則請讀取 `config/AI_CODING_RULES.md`。
