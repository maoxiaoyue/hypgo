# 0.4版想法
## 1.plugin在Go的機制中實在多餘
其實，Go本身就已經很彈性，在main裡就可以自己加東西，也不知道是中邪、還是PHP的思維影響的，搞個plugin，多餘!

所以，就花了時間把它抽掉，減少這個框架的資源消耗

## 2.HTTP/3.0 有點麻煩
之前，在Gin裡面直接用quic-go，問題不大，但寫成框架要利用config來設定，搞的就有一點麻煩，因為，他走的是UDP，

而非以往的TCP，很多機制要定義好...一個設定就要弄好所有機制，包含Router、Middleware跟Context就會糾纏不清...

## 3.Gin已加入HTTP/3實驗性支援
Gin的v1.11.0也加入此功能，一樣是用go-quic來完成的，雖然是實驗性，這也表示未來會完全支援，實在大大的打擊
